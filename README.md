<h1 align="center">module_Document_Reader</h1>

## описание:
Модуль читает работы студентов и при помощи NLP модели spaCy извлекает ФИО преподавателей и студентов

### Описание основных функций модуля

1. `__init__()` - инициализация объекта MDR. Не принимает никакие аргументы, но проверяет наличие NLP модели 'ru_core_news_lg'. В случае отсутсвия будет выведено следующее сообщение:
```
Error: spaCy model 'ru_core_news_lg' not found.
Please run: python -m spacy download ru_core_news_lg
```

2. `read_document(path_: str)` - чтение документа, лежащего по пути `path_`. Если в переменную было передано только название файла, то модуль будет считать, что он находится в той же директории, что и модуль - то есть в `/mdr/<название файла>`. Функция определяет наличие полного пути тем, что в строке присутствует `/`.

    Важно: на данный момент модуль умеет работать только с документами формата .doc и .docx. В случае попытки чтения файла иного формата будет выведена ошибка чтения:

    `ERROR: Не удалось найти файл!`

3. `extract_names()` - нахождение имён преподавателей и студентов работы из прочитанного документа. Поиск осуществляется при помощи NLP библиотеки spaCy и её модуля под русский язык - `ru_core_news_lg`. Поиск происходит при помощи **spaCy.Matcher**, который осуществляет его по шаблону, который выглядит для поиска преподавателей так:

    ``
    pattern_tc = [...]
    ``

    для студентов так:

    ``
    pattern_st = [...]
    ``

    `TEACHERS_IDENTIFICATIONS_WORDS` и `STUDENTS_IDENTIFICATIONS_WORDS` - массивы с ключевыми словами, которые должны быть в тексте, чтобы шаблон смог сработать. Слова в нём могут находится лишь в одной форме - например, для слова "выполнил" не нужно будет перебирать все его временные формы и под каждый пол, нет, нужно просто ввести его начальную форму в массив и модель `ru_core_news_lg` сама сможет находить все его форму. 
    
    ! Важная оговорка, иногда модель даёт сбой и всё равно прирходится вписывать некоторые слова. Вполне возможно, что это является следствием моей ошибки, поэтому я постараюсь решить эту проблему.

    После завершения поиска в консоль будут выведены списоки студентов и преподавателей

### Функции получения данных (getter'ы)

a. `get_students() -> list` - получение массива студентов-авторов документа, где каждый студент представляет собой отдельный массив с найденными именами.

b. `get_teachers() -> list` - получение массива преподавателей, где каждый преподаватель представляется, как отдельный массив с найденными именами.

c. `get_doc_name() -> str` - вывод названия прочитанного документа.
